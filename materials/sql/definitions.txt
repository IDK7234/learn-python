1.База данных (БД) — это упорядоченный набор информации, который хранится в электронном виде.

2.СУБД — это аббревиатура от «Система управления базами данных». 12 Так называют программное обеспечение, 
которое позволяет создавать, изменять, администрировать базы данных.

3.Запрос SQL— это запрос, выполняемый пользователем или приложением для извлечения данных, хранящихся в базе данных, 
или манипулирования ими.

4.SQL (от англ. Structured Query Language — «язык структурированных запросов») — это декларативный язык программирования, 
который используют для создания, обработки и хранения данных в реляционных базах данных.

5.Структурированный запрос - это запрос, который имеет чёткую структуру.

6.Реляционный — relational произошло от слова relation, что означает отношение

7.Реляционная модель данных - это модель представления и организации данных в базе данных. В реляционной модели данные 
хранятся в виде таблиц, состоящих из строк и столбцов. Каждая таблица имеет имя и структуру, определяемую набором столбцов и их
типами данных. И в этой модели данные из одной таблицы могут зависеть от данных в другой таблице.

8.Таблица - основной объект в БД. Она состоит из строк, столбцов и полей. Строки - это набор данных по горизонтали.
Столбцы - это набор данных по вертикали. Поля - это названия столбцов.

9.Столбец - это одно поле в таблице, которое хранит данные одного типа. Каждый столбец имеет уникальное имя и тип данных, 
который определяет, какой тип данных может храниться в столбце.

10.Строка - это одна запись в таблице, которая содержит данные для каждого столбца. Каждая строка в таблице имеет уникальный 
идентификатор, который называется ключом. Ключ может быть составным, т.е. может включать несколько столбцов.

11.Ключ - это уникальный идентификатор для каждой записи в таблице. Ключ может быть составным, т.е. может включать несколько 
столбцов, которые вместе обеспечивают уникальность записи в таблице. Ключ используется для связывания данных в разных таблицах и 
для обеспечения быстрого доступа к данным. (id)
